(1)去掉风险分析功能里的混合分析，改为智能分析，下面是智能分析的设计方案
第一步：本地 YARA 极速前置扫描 (Local Pre-scan)
当样本在第一阶段（快速查询）未命中黑白名单，进入智能分析阶段时，先不急着调云端 API，而是立即在 EDR 客户端或本地服务器执行内存/文件 YARA 扫描。

根据 YARA 规则的命中情况，将样本打上特征标签：

高确信度规则命中（例：包含 Cobalt Strike 典型的配置明文、勒索软件专属加密扩展名）：直接判定高危，终止流程，不仅省下一次 API 调用，还能实现毫秒级拦截。

行为特征/混淆特征命中（例：使用了高度混淆的 PowerShell 语法、包含虚拟化逃逸 API 引入、UPX 加壳）：标记为“可疑”，进入云端融合流。

未命中任何规则：标记为“无明显静态特征”，进入常规云端排队流。

第二步：基于 YARA 标签的“定向 API 调用”
这是对抗“API 限流”最有效的方法。根据 YARA 扫描出的标签，智能决定调用哪个平台的 API，而不是盲目群发。

场景 A：YARA 提示样本是“宏文档 (Office Macro)”或“PowerShell 脚本”。

动作： 直接放弃查询 VirusTotal 的基础库（因为静态引擎对混淆脚本效果差），定向将 Hash 甚至文件提交给 Hybrid Analysis。因为 HA 对脚本的反混淆和执行链解析是最强的。

场景 B：YARA 提示样本包含了“可疑的远控网络协议字段”或“特定 APT 工具的弱特征”。

动作： 定向请求 AlienVault OTX。因为这类样本往往关联着特定的黑客组织，OTX 的 Pulse（脉冲）能迅速告诉你这个特征是否属于某个正在活跃的 APT 团伙。

场景 C：YARA 提示“仅是一个被加壳的未知 PE 文件”。

动作： 请求 VirusTotal (Relations)，不看查杀率，只看这个文件释放了什么 (Dropped Files) 或者联了什么 IP。

第三步：端云交叉验证的归一化评分 (Cross-Validation Scoring)
将本地 YARA 的特征权重与云端 API 返回的上下文数据进行数学融合。可以采用一个基础的计分矩阵：

本地 YARA 基础分 (0-40分)：

命中通用加壳/加密特征：+10分

命中反调试/反沙箱特征：+20分

命中已知黑客工具链弱特征：+30分

云端情报附加分 (0-60分)：

OTX 发现关联的可疑 IP：+20分

VT 报毒数 1-5 个（灰度）：+15分

HA 的威胁评分映射：按比例转化为 0-30 分

融合调节系数 (Multiplier)：

如果 YARA 命中的特征与 OTX 返回的攻击组织 TTPs 逻辑吻合（例如 YARA 扫出钓鱼邮件特征，OTX 显示该 Hash 关联了钓鱼活动），总分额外 * 1.2（最高不超过 100 分）。

判定阈值：

综合得分 >= 80：视为高危恶意文件，立即阻断并隔离。

50 <= 综合得分 < 80：视为高度可疑，不杀文件，但限制其网络访问权限，并推入第三阶段（深度沙箱分析）。

综合得分 < 50：视为安全/未知，放行但保持本地行为监控。

(2)针对联网分析功能进行更改，分为快速分析，智能分析，深度分析三种调用方式
🛠️ 第一阶段：快速分析 (Fast Lookup)定位： 秒级响应，高确信度过滤。解决 80% 的已知黑白样本。触发条件： EDR 捕获到新进程启动或新文件落地，且本地缓存未命中。调用工具： MalwareBazaar + VirusTotal (仅限 Hash 查询)执行逻辑：首选 MalwareBazaar： 并发查询 MB。因为它“非黑即白”，只要命中，提取 signature 和 tags，直接下发高危拦截指令并标记家族，流程结束。次选 VT 基础查询： 如果 MB 未命中，查询 VT 的 Hash 报告。归一化与权重：如果 VT 报毒数 $Positives \ge 10$，且包含微软、卡巴斯基等大厂，判定为高危。如果 $0 < Positives < 5$ 且多为不知名引擎，判定为低危/可疑，进入下一阶段。防限流策略： 针对 VT 免费版（4次/分钟），必须在代码中实现 Token Bucket（令牌桶）算法或简单的 Redis 限速锁。超限的请求进入挂起队列，或者直接暂时返回“未知”，降级到本地规则处理，绝不死磕等待。🧠 第二阶段：智能分析 (Contextual Intelligence)定位： 分钟级响应，上下文关联。处理在第一阶段表现为“灰度”（如仅有 1-3 个小引擎报毒）的样本。触发条件： 快速分析无法一锤定音，但样本在本地有敏感行为（如注册表修改、网络外连）。调用工具： AlienVault OTX + VirusTotal (Relations 图谱)执行逻辑：查询 OTX 脉冲： 将 Hash 投入 OTX 查询关联的 Pulse。信誉降噪： 严格解析 API 返回的 Pulse 作者信息。如果作者是官方实验室 (Alien Labs) 或信誉分极高的研究员，且标签涉及特定 APT 组织，直接升级为高危。如果是普通用户提交，仅作打分参考。VT 图谱关联： 获取 VT 的 Relations 数据，提取该 Hash 通信的 IP/URL。如果这些 IP 命中了本地的威胁情报库，判定为恶意。归一化与权重： 这一步可以采用加权计分公式，例如：$Risk\_Score = (VT\_Score \times 0.3) + (OTX\_Trusted\_Pulse \times 0.7)$当总分超过设定阈值时，触发 EDR 告警。🔬 第三阶段：深度分析 (Deep Dynamic Analysis)定位： 耗时数分钟到十分钟，深度解剖。处理高伪装、加壳、或针对性攻击的未知样本。触发条件： 属于高危文件类型（如 Office 宏文档、PDF、未知签名的 PE 文件），且前两阶段均无结果，但 EDR 本地行为引擎判定其“高度可疑”。调用工具： Triage (主力首选) + Hybrid Analysis (脚本/文档备选)执行逻辑：按类型分发上传： 将 PE/可执行文件推给 Triage（速度快，并发好）；将 Office 文档和 PowerShell 脚本推给 Hybrid Analysis（文档解析极强）。Webhook 回调等待： 沙箱分析时间长，不要用循环轮询 (Polling)，而是利用 API 提交时配置 Webhook URL，让沙箱分析完主动把报告推给你的服务器。核心数据提取：Triage： 重点提取 extracted 字段。如果拿到 C2 域名或 IP，立即联动 EDR 下发网络阻断策略。提取评分（$\ge 8$ 判黑）。HA： 重点提取 MITRE ATT&CK T-Codes。如果报告中出现了 $T1055$ (进程注入) 或 $T1003$ (凭证获取) 等高危战术，结合其 $threat\_score$，综合判定风险。

1. 快速分析 (Fast Analysis) 的超时与回退
定位： 主线程阻塞阶段，用户双击程序，系统等待放行。

超时限制 (Hard Timeout)： 2 秒。

如果请求 VirusTotal / MalwareBazaar 接口超过 2 秒未拿到 HTTP 200 返回，立即强行切断网络请求。

回退机制：YARA-X 极速扫描 (Fast Scan)

触发动作： 云端超时断开的瞬间，立刻在本地调用 YARA-X 引擎。

规则策略 (仅限 High-Confidence 规则)： 在这个阶段，决不能加载几万条庞大的社区规则。只加载本地缓存的**“高确信度、无通配符、纯强特征”**规则集（例如：已知的勒索病毒特定文件头、Cobalt Strike 的固定明文特征）。

本地执行时间预算： < 500 毫秒。

管控结果： 如果 YARA-X 极速扫描命中，直接拦截；如果未命中，放行进程，将云端查询任务推入后台重试队列。

2. 智能分析 (Smart Analysis) 的超时与回退
定位： 进程已在运行（受控状态），云端在拉取 OTX 或 VT 关系图谱等上下文情报。

超时限制 (Soft Timeout)： 10 秒。

服务端推送到消息队列的任务，设置 10 秒的 TTL（存活期）。如果 10 秒内云端情报 API 没回包（限流或网络抖动），任务过期。

回退机制：YARA-X 深度静态启发扫描 (Heuristic Scan)

触发动作： 既然云端无法提供关联情报，我们必须榨干样本本地的静态特征。

规则策略 (启发式与混淆规则)： 加载耗时相对较长的规则集。重点扫描样本是否包含可疑的 API 导入表（如 VirtualAlloc + CreateRemoteThread 的组合）、高熵值（判断是否加壳）、或者混淆的 PowerShell/VBA 脚本特征。

本地执行时间预算： 1 秒 - 3 秒。

管控结果： 结合 YARA-X 扫出的“可疑标签数量”进行本地打分。如果达到中高危阈值，立刻下发指令，限制该进程的本地权限和网络访问（微隔离）。

3. 深度分析 (Deep Analysis) 的超时与回退
定位： 云端沙箱跑测。文件已经上传，系统在等待动态行为报告。此时进程在本地可能已经存活了十几分钟。

超时限制 (Async Timeout)： 15 分钟。

如果 Triage 或 Hybrid Analysis 在 15 分钟内没有通过 Webhook 推送分析报告，或者轮询状态一直卡死，标记该云端任务失败。

回退机制：YARA-X 进程内存扫描 (Memory Scan)

触发动作： 15 分钟过去了，如果样本是恶意的，它极大概率已经在内存中完成了解密和脱壳。此时再去扫磁盘上的源文件毫无意义。

规则策略 (内存抓取与特征匹配)： EDR 客户端接到超时的降级指令后，提取该进程当前在系统中的内存 Dump（或者直接对运行中的进程空间使用 YARA-X 进行扫描）。使用包含 C2 域名正则、内存注入特征的规则集去匹配。

管控结果： 内存扫描往往能抓到最真实的恶意特征（如解密后的木马配置）。如果命中，不仅要 Kill 掉该进程，还要顺藤摸瓜清理它创建的子进程和注册表项。如果未命中，生成告警转交人工 SOC 分析。
