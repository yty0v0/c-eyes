## Why

需要在 EDR 中提供文件扫描能力，覆盖全盘与指定路径并引入低资源消耗的智能扫描，以便快速发现高危入侵路径与近期活跃威胁，补足仅进程扫描的检测空白。

## What Changes

- 新增文件扫描能力，支持全盘扫描、指定路径扫描、智能扫描三种模式。
- 智能扫描采用“目标收集 -> 过滤引擎 -> 深度扫描 -> 结果上报”的管线，并约束资源占用。
- 增加本地扫描缓存（SQLite）与云端信誉查询流程，支持黑白名单短路。
- 增加调度触发（系统空闲/行为驱动）与 ScanTask 入口。
- 结果输出支持 JSON 与 Excel 两种格式，并提供请求/返回参数定义。

## Capabilities

### New Capabilities
- `file-scan`: EDR 文件扫描能力，覆盖三种扫描模式与智能扫描管线、缓存与结果输出规范。

### Modified Capabilities
- （无）

## Impact

- CLI/接口层新增文件扫描命令与参数校验。
- 新增/修改扫描调度器、目标收集、过滤、深度扫描、结果上报模块。
- 新增本地缓存表与云端信誉查询依赖。
- 新增 JSON/Excel 导出逻辑与测试覆盖。
