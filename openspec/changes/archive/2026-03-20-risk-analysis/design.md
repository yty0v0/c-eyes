## Context

现有扫描流程输出原始结果，但缺少统一的风险分析与评分，无法直接用于处置与告警。需求要求基于扫描结果文件进行分析，支持本地 YARA-X 与云端威胁情报两种模式，并输出 JSON/Excel。

## Goals / Non-Goals

**Goals:**
- 增加风险分析模块，基于扫描结果文件生成标准化风险结论与评分。
- 提供本地匹配（YARA-X）、云端查询（如 VirusTotal）及混合模式。
- 评分结果映射为无/低/中/高风险，并支持 JSON/Excel 输出。
- 保持对扫描结果字段的扩展兼容（字段可增量扩展）。

**Non-Goals:**
- 不改变现有扫描器的检测逻辑与规则语义。
- 不上传文件本体到云端（仅使用哈希与必要元数据）。
- 不在本次变更中定义完整 UI/告警策略。

## Decisions

- **输入格式与解析**：以“扫描结果文件”为唯一输入，优先支持 JSON。为兼容字段扩展，核心字段采用强类型结构体，非关键字段保留 `map[string]any` 以透传。
  - 备选：强制固定 schema。放弃原因：需求明确字段在持续扩展。
  - 补充：风险分析已支持 Excel 输入（扫描结果表格），并兼容 JSON 数组/NDJSON。

- **分析模式与参数**：提供 `local_only`、`cloud_only`、`hybrid` 三种模式，通过独立参数启用；混合模式执行两条链路并加权汇总。
  - 备选：自动模式判断。放弃原因：用户明确“对应单独参数来调用”。
  - 补充：新增输入源参数 `-file`、`-dir -r`、`-pid`、`-pname`，并强制输入源互斥。

- **本地 YARA-X 集成**：采用嵌入式库调用方式集成 `yara-x`，通过 cgo 绑定其 C API 或静态库；规则文件支持预编译与热加载。
  - 备选：外部进程调用。放弃原因：不符合“嵌入式库调用”。
  - 补充：`local_only` 下 YARA-X 不可用直接失败；`hybrid` 下自动降级为 `cloud_only` 并输出 warning。
  - 补充：新增跨主机误扫防护（hostname/hash 校验），不满足身份校验时本地匹配回退并写明原因。
  - 补充：新增 `process_memory` 本地分析路径（可选），并引入按块读取配置 `-yara-read-chunk`。

- **云端查询**：采用多平台并行聚合（VirusTotal、Hybrid Analysis、MalwareBazaar、OTX、Triage），以哈希为主查询；加入缓存与限频控制，平台失败时局部降级并在聚合层容错。
  - 备选：单平台查询。放弃原因：覆盖率与鲁棒性不足。
  - 补充：云端连接参数（API key/base URL/proxy/rate limit/timeout/cache TTL）统一配置化，支持全局代理与平台代理覆盖。
  - 补充：代理仅作用于联网查询，不影响本地 YARA 分析。
  - 补充：单平台入口保留兼容但标记废弃（`-cloud-provider`/`EDR_CLOUD_PROVIDER`），最终执行固定并行聚合。
  - 补充：聚合结果使用统一标识 `cloud_provider=multi`，并输出 `cloud_providers` 记录有效命中平台。

- **配置文件发现策略**：云端配置文件 `edr-cloud.json` 采用固定查找顺序：`EDR_CLOUD_CONFIG` -> 可执行文件同目录 -> 当前目录 -> `~/.edr/cloud.json`。
  - 备选：仅环境变量注入。放弃原因：分发和运维成本高，不利于可执行文件旁路配置。

- **本地规则路径策略**：本地模式规则路径按优先级解析：`-yara-rules` -> `EDR_YARA_RULES` -> 可执行文件同目录 `rules/yaraRules` 或 `rules`。
  - 备选：强制每次传参。放弃原因：对打包交付不友好。

- **分发与打包策略**：发布目录 `dist` 打包可执行文件 + YARA-X 动态库 + 规则目录，`edr-cloud.json` 不强制打包，允许与可执行文件同目录独立部署。
  - 备选：将 API key 编译进可执行文件。放弃原因：安全风险高且更新成本高。

- **评分策略**：本地与云端结果各自产生分值，按权重合成最终 `risk_score`，再按阈值映射为风险等级（0-20 无、21-50 低、51-80 中、81-100 高），分值裁剪到 0-100。
  - 备选：仅云端或仅本地单分值。放弃原因：无法体现双模式优势。
  - 补充：云端聚合采用平台差异化权重，并对高置信平台命中应用分值下限策略（MalwareBazaar/Triage）。

- **输出格式**：JSON 为默认输出；Excel 使用 `excelize` 生成单表或多表（按扫描对象/模式拆分），字段与 JSON 保持一致。
  - 备选：CSV。放弃原因：不满足 Excel 要求。

## Risks / Trade-offs

- **[CGO/构建复杂度]** → 需要处理跨平台编译与依赖分发，提供默认构建脚本与文档。
- **[云端 API 限频/费用]** → 引入速率限制、缓存与可配置开关，必要时降级仅本地。
- **[扫描结果字段不稳定]** → 采用容错解析与字段透传，关键字段缺失时输出显式错误。
- **[性能开销]** → 本地规则匹配与云端请求并行执行，避免阻塞主扫描流程。
- **[跨主机误判]** → 增加 hostname/hash 前置校验与 fallback 标记，避免“同路径误扫本机文件”。
- **[进程内存权限限制]** → 内存采集失败按记录回退，不阻断整体任务。
- **[配置漂移风险]** → 配置文件与可执行文件分发分离可能导致版本不一致；通过固定路径查找顺序和模板约定降低误配。

## Migration Plan

1. 新增风险分析模块与配置项，默认关闭。
2. 接入本地 YARA-X 与云端查询客户端并实现评分策略。
3. 增加 JSON/Excel 输出与集成测试样例。
4. 文档与示例更新，灰度启用。
