## Context

Web 资产采集已具备静态解析与动态关联能力，但在 include 配置链、软链接配置路径与复杂启动参数场景下仍可能出现漏采或去重不稳。为保证跨环境一致性，需要提升解析器与路径归一化策略。

## Goals / Non-Goals

**Goals:**
- 递归解析 Nginx/Apache include 配置链，并设置最大深度保护。
- 将配置路径统一做软链接真实路径解析后再参与合并与去重。
- 输出增加 `isRunning` 字段，明确资产活跃状态。
- 增加边界测试覆盖复杂参数、相对路径、合并去重。

**Non-Goals:**
- 不扩展风险分析能力。
- 不引入外部命令采集路径。

## Decisions

1. include 解析采用“受限递归 + 防循环 visited”。  
理由：提升覆盖率同时避免 include 递归陷阱。

2. 路径归一化统一使用 `EvalSymlinks + Clean`。  
理由：确保去重维度稳定，避免软链接路径造成重复资产。

3. `isRunning` 默认 false，动态命中后置 true。  
理由：下游可直接区分活跃资产与静态遗留配置。

4. Excel 同步新增 `isRunning` 列。  
理由：保持 JSON/Excel 契约一致。

## Risks / Trade-offs

- [include 递归成本增加] → 限制最大深度与去重集合。
- [软链接解析在权限受限目录失败] → 失败时回退到原路径，不中断采集流程。
- [新增字段影响下游解析] → 字段追加不破坏既有键语义，并通过测试校验。
