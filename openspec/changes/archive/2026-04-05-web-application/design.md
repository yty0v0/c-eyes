## Context

当前 EDR 缺少针对 Web 应用资产的统一采集能力，导致 Windows/Linux 两端的数据字段不一致、查询维度不统一、导出链路重复建设。该变更要求以 CLI 触发，但采集过程不得依赖外部命令行工具，仅进行信息收集，不包含风险分析结论。

输入侧需要支持业务组、主机标识、应用标识、目录路径、服务类型和域名等过滤参数；输出侧需要提供稳定字段契约，并同时支持 JSON 与 Excel 导出。另一个关键约束是主机内外网 IP 需要以数组表达，覆盖多网卡场景。

## Goals / Non-Goals

**Goals:**
- 提供统一的 `web-application-scan` 能力，在 Windows 与 Linux 返回一致的数据契约。
- 支持请求参数 `groups`、`hostname`、`ip`、`version`、`appName`、`rootPath`、`webRoot`、`serverName`、`domainName`。
- 支持应用与插件信息采集，并输出稳定字段结构。
- 支持 JSON 标准输出和 Excel 导出两种结果格式。
- 明确能力边界为“信息采集与规范化”，不输出风险评分或威胁结论。

**Non-Goals:**
- 不在本能力中实现风险判定、告警聚合、处置建议。
- 不引入远程主动探测或漏洞扫描逻辑。
- 不改变现有其它扫描能力的输出契约。

## Decisions

1. 采用“统一领域模型 + 平台采集适配器”
- 决策：定义统一 `WebApplicationRecord` 与 `PluginRecord` 模型；Windows/Linux 分别实现采集适配器并汇总到统一模型。
- 原因：保持跨平台字段一致，减少过滤、导出、测试逻辑分叉。
- 备选方案：平台各自输出不同结构；被拒绝，因为会增加 CLI 与下游消费复杂度。

2. 采集路径必须为进程内 API/数据源解析
- 决策：Windows 通过系统 API、注册表、配置文件读取等方式采集；Linux 通过配置目录与文件解析采集；禁止通过外部命令执行获取数据。
- 原因：满足“非命令行采集”约束，减少环境依赖与命令注入面。
- 备选方案：使用 `nginx -V`、`apachectl -v` 等命令抓取；被拒绝，违反约束。

3. 过滤策略在标准化之后执行
- 决策：先完成跨平台数据标准化，再执行组合过滤（模糊与结构化参数）。
- 原因：确保过滤语义一致，不受平台原始字段差异影响。
- 备选方案：平台侧先过滤；被拒绝，因为跨平台行为难保持一致。

4. 输出层采用 Writer 抽象
- 决策：定义统一 writer 接口，至少实现 `JSONWriter` 与 `ExcelWriter`。
- 原因：降低采集逻辑与导出格式耦合，便于后续扩展 CSV/数据库写入等格式。
- 备选方案：在 CLI 命令层直接分支输出；被拒绝，因可维护性和可测试性较差。

5. IP 字段统一为显示字段 + 列表字段
- 决策：保留 `displayIp` 作为主显示字段，同时输出 `internalIpList` 与 `externalIpList` 作为完整地址集合。
- 原因：满足多网卡主机全量采集要求，并与现有扫描能力的数组语义保持一致。
- 备选方案：仅输出单值内外网 IP；被拒绝，因为会丢失信息。

## Risks / Trade-offs

- [Risk] Linux 发行版及安装路径差异导致 Web 根目录漏采集 -> Mitigation: 维护可配置的路径探测清单并补充样本测试。
- [Risk] 插件元数据来源不统一（不同框架字段差异大） -> Mitigation: 定义最小公共字段并对缺失字段使用 null/空值回填。
- [Risk] Excel 导出在大数据量下内存占用偏高 -> Mitigation: 使用流式写入并增加单次导出上限保护。
- [Risk] 版本识别规则误报/漏报 -> Mitigation: 采用多来源交叉匹配（配置、目录、元文件）并记录原始来源字段用于排障。

## Migration Plan

1. 注册 `edr web-application-scan` 子命令并接入参数解析与校验。
2. 建立统一记录模型与过滤管线，接入主机信息补全与 IP 列表聚合。
3. 实现 Windows/Linux 平台采集器，保证采集过程不启动外部命令。
4. 实现 JSON/Excel writer，并对输出字段稳定性做一致性验证。
5. 增加端到端测试，确认输出仅包含信息采集字段，不包含风险分析字段。

回滚策略：
- 若新能力存在兼容性问题，可在命令注册层临时下线 `web-application-scan`，不影响现有扫描命令。

## Open Questions

- `serverName` 枚举是否仅限 `nginx/apache/tomcat`，还是允许扩展值透传？
- Excel 导出的列名是否与 JSON 字段完全同名，还是需要业务侧中文列头映射？
- `plugins` 字段在 Excel 中采用 JSON 字符串单列，还是拆分为多列/多表结构？
