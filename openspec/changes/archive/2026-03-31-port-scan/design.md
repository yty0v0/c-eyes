## Context

现有仓库已具备多类资产扫描能力，但缺少统一的端口信息扫描模块。`docs/port-scan.md` 约束了跨平台（Windows/Linux）、采集方式（不得调用外部命令）、扫描模式（`tcp-connect` 与 `tcp-syn`）和输出格式（JSON/Excel），并明确该能力仅用于信息收集，不包含风险分析。设计需要在复用现有 CLI 和导出框架的前提下，新增可独立演进的端口扫描能力并保持字段契约稳定。

## Goals / Non-Goals

**Goals:**
- 提供 `edr port-scan` 命令，默认 `tcp-connect`，可切换 `tcp-syn`。
- 支持 `groups`、`hostname`、`ip`、`proto`、`port`、`bindIp`、`processName` 过滤参数。
- Windows/Linux 均通过进程内实现采集，不依赖外部命令行工具。
- 统一输出字段和空值策略，支持 JSON/Excel 导出。
- 保持与既有资产扫描一致的命令体验和返回结构。

**Non-Goals:**
- 不实现端口风险分级、风险评分、告警或阻断。
- 不实现远程分布式扫描编排，仅处理本机可获取信息。
- 不在本变更引入新的云侧 API 协议。

## Decisions

1. 新增独立模块 `internal/portscan`
   - 方案：创建 `types.go`、`scan.go`、`filter.go`、`scan_linux.go`、`scan_windows.go`，并在 CLI 层接入 `port-scan` 子命令。
   - 原因：与现有扫描能力模块化方式一致，降低耦合和维护成本。
   - 备选：扩展已有 `process-info-scan` 模块。
   - 未选原因：端口扫描有独立扫描模式与字段契约，混合实现会导致职责不清。

2. 扫描模式统一抽象为策略接口
   - 方案：定义 `ScanMode`（`tcp-connect`/`tcp-syn`）并通过统一入口调度不同探测策略，默认 `tcp-connect`。其中 `tcp-connect` 在采集结果上增加一次连接可达性验证，`tcp-syn` 保持被动采集（不建立完整连接）。
   - 原因：保证默认行为稳定，同时便于后续扩展 UDP 等模式。
   - 备选：在单个函数中用分支实现所有模式。
   - 未选原因：可维护性差，测试隔离难。

3. 平台采集全部采用进程内系统能力
   - Linux：基于 `/proc`、socket 与系统接口映射端口-进程关系，不执行外部命令。
   - Windows：基于系统网络与进程 API 获取监听/连接端口及进程信息，不执行外部命令。
   - 原因：满足“禁止命令行采集”的硬约束，并提升可控性。
   - 备选：调用 `netstat`、`ss` 等系统命令解析输出。
   - 未选原因：违反需求约束，且输出格式脆弱。

4. 输出模型固定字段 + 平台差异兜底
   - 方案：输出结构始终包含需求定义字段；不可获取字段返回 `null`（或空数组）但不缺失字段键。
   - 原因：减少消费端分支逻辑，保持跨平台兼容。
   - 备选：按平台裁剪字段。
   - 未选原因：破坏接口稳定性。

5. 导出复用现有输出管线
   - 方案：在统一输出层新增端口扫描 Excel 列映射，沿用 `-output json|excel` 语义。
   - 原因：降低重复实现和学习成本。
   - 备选：独立实现导出命令。
   - 未选原因：增加认知与维护成本。

## Risks / Trade-offs

- [tcp-connect 模式在本机防火墙或策略限制下可能出现连接验证失败] -> 失败时保留基础采集状态映射并保留端口原始记录供排查。
- [不同平台端口状态语义不完全一致] -> 统一映射到 `-1/0/1`，未知状态返回 `-1`。
- [高端口范围扫描可能耗时较长] -> 通过并发上限与超时配置控制资源占用。
- [端口到进程映射可能存在瞬时不一致] -> 采集时记录快照时间并在文档标注“近实时”语义。

## Migration Plan

- 第一步：新增 `internal/portscan` 数据模型和跨平台扫描骨架。
- 第二步：接入 CLI 参数解析、模式选择和过滤逻辑。
- 第三步：接入 JSON/Excel 导出与字段映射，补齐帮助文档。
- 第四步：补充单测/集成测试并在双平台回归验证。
- 回滚策略：若出现兼容问题，移除 `port-scan` 子命令和模块接入，不影响既有扫描能力。

## Open Questions

- `status` 字段中“内网/外网可访问”在离线主机场景的判定边界是否需要更细规则。
- `proto` 参数是否需要限制为固定枚举（如 `tcp`）还是保留向后扩展空间。
- Excel 导出是否需要支持自定义列选择（本次默认全字段导出）。
