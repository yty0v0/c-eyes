## Context

当前 EDR 缺少面向 Jar 包的统一资产采集能力，导致不同主机上的应用依赖、系统依赖与运行组件无法标准化汇总。该能力要求在 Windows 与 Linux 端都可运行，通过 CLI 触发执行，但采集阶段不能依赖外部命令，仅做信息收集，不进入风险判断。

需求侧已明确过滤参数（`groups`、`hostname`、`ip`、`name`、`version`、`type`、`executable`、`path`）和返回契约（主机字段 + Jar 字段），并要求将内外网地址改为数组化全量采集，输出 JSON/Excel 两种格式。

## Goals / Non-Goals

**Goals:**
- 提供 `jar-package-scan` 跨平台能力，在 Windows/Linux 输出一致字段结构。
- 采用“静态 + 动态”结合方式提升 Jar 识别覆盖率。
- 保证采集过程不调用外部命令行命令。
- 支持过滤参数组合查询与稳定的结果契约。
- 支持 JSON 与 Excel 导出，内容与字段映射保持一致。
- 使用 `displayIp` + `internalIpList`/`externalIpList` 形式覆盖多网卡场景。

**Non-Goals:**
- 不输出风险评分、漏洞结论、威胁等级等风险分析字段。
- 不在本变更中改造其他扫描命令的契约与业务流程。
- 不引入主动探测或远程网络扫描行为。

## Decisions

1. 统一领域模型
- 决策：定义统一 `JarPackageRecord` 模型，包含主机元信息和 Jar 包字段（`name`、`version`、`type`、`executable`、`path`）。
- 原因：减少平台分叉，统一过滤、导出与测试逻辑。
- 备选方案：Windows/Linux 分别建模后在导出层做兼容；被拒绝，因字段漂移风险和维护成本更高。

2. 静态 + 动态双通道采集
- 决策：静态通道扫描部署目录与元数据文件（如 jar 名称、清单、目录结构）；动态通道关联运行态映射（进程加载路径、应用工作目录等），最终去重合并。
- 原因：单一静态或动态方案会导致漏采，双通道更稳健。
- 备选方案：仅静态目录扫描；被拒绝，无法覆盖运行时路径与挂载目录变化。

3. 非命令行采集约束
- 决策：采集仅使用进程内 API、文件系统读取、配置解析和已有运行态数据，不启动外部命令。
- 原因：满足安全约束与跨平台一致性要求。
- 备选方案：依赖 `ps`、`wmic`、`find` 等命令；被拒绝，违反输入约束并引入环境依赖。

4. 过滤器标准化
- 决策：统一过滤管线，先做参数校验，再执行模糊匹配（`hostname`、`ip`、`name`、`path`）和结构匹配（`groups`、`version`、`type`、`executable`）的组合过滤。
- 原因：保证命令行为可预测，便于测试覆盖。
- 备选方案：按平台分支各自过滤；被拒绝，因结果一致性差。

5. 导出层 Writer 抽象
- 决策：复用统一导出接口，提供 JSON Writer 与 Excel Writer，共享同一字段映射与空值回退策略。
- 原因：保证格式一致，减少重复逻辑。
- 备选方案：在命令层分别组装 JSON/Excel；被拒绝，因扩展性差且易出现字段不一致。

## Risks / Trade-offs

- [Risk] Jar 版本解析在非标准命名包上可能不准确。 -> Mitigation: 优先读取 manifest 元数据，失败时回退文件名规则并记录空值。
- [Risk] 动态关联可能把同主机多应用的 Jar 归属混淆。 -> Mitigation: 使用多键匹配（进程路径、工作目录、部署根目录）并加入冲突回退策略。
- [Risk] 大规模 Jar 清单会增加 Excel 导出体积和内存压力。 -> Mitigation: 采用流式写入与分页/条数上限策略。
- [Risk] 下游若仍依赖单值 `internalIp`/`externalIp` 会出现兼容问题。 -> Mitigation: 在发布说明提供映射示例并同步字段变更公告。

## Migration Plan

1. 增加 `jar-package-scan` 命令入口及参数解析与校验。
2. 建立统一 `JarPackageRecord` 模型并实现过滤管线。
3. 接入 Windows/Linux 采集适配器（静态 + 动态），并完成去重合并。
4. 实现 JSON/Excel 导出并统一字段映射、UTF-8 输出。
5. 增加单元测试与集成测试，验证过滤行为、字段稳定性、导出一致性和“无风险字段”约束。

回滚策略：
- 如上线后出现兼容性问题，可先在命令注册层临时下线 `jar-package-scan` 子命令，不影响现有已上线扫描能力。

## Open Questions

- `type` 字段是否固定枚举（`1/2/3/8`）并在输出中映射可读文案，还是允许扩展值透传？
- `version` 查询参数是单值还是多值数组（需求材料存在两种表述）？
- Excel 中 `hostTagList` 与 IP 列表字段采用逗号拼接还是 JSON 字符串持久化？
